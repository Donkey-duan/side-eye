# side-eye

台灣性別事件公共資訊平台。彙整政府開放統計、教育部公開名冊、司法院裁判書與公共新聞，
並整理校園性平與刑事程序的實務流程。

只收錄官方公開來源；未接上的來源標示現況，不以示意資料填補。無個別事件的跨來源索引，
不提供人物姓名查詢。

## 架構

純靜態站。資料由 GitHub Actions 每晚產生為 JSON，隨網站一併發布至 GitHub Pages；
搜尋與篩選在瀏覽器端完成，無伺服器亦無資料庫。

```
每晚 UTC 17:00（臺北 01:00）
  ├─ 公視新聞搜尋 : 逐監測詞查詢，與前一版索引合併累積
  ├─ 司法院裁判書異動清單（API 僅該時段開放）
  ├─ 教育部性平人才名冊
  └─ next build → 26 個靜態頁面 → Pages
```

累積機制以 `SITE_URL` 取回已發布的前一版再合併，因此索引隨時間增長。
資料來源異常時工作流程直接失敗，避免「照常發布但資料未更新」的無聲失效。

### 裁判資料的兩段式流程

抽取與分類分開。抽取（解壓月包、解析全文）耗時且結果不變，分類與統計則會隨判準
調整而重算，混在一起會導致改一條規則就要重跑二十五分鐘。

```
官方月包（RAR）→ import-judgments.mjs → cache/judgments.jsonl（本機，不進版控）
                                          ↓
每晚 API 異動清單 ────────────────→ analyse-judgments.mjs
                                          ↓
              judgments.json（逐筆紀錄）· judicial.json（統計摘要）· judicial-judges.json（法官明細）
```

`judgments.json` 是累加與驗算的依據：每晚同步讀回前一版再併入新的異動，
任何人也可以下載它自行驗證統計數字。瀏覽器只載入摘要，不載入逐筆紀錄。
判別規則集中在 `scripts/judgment-rules.mjs`，兩個來源共用同一份。

## 路由

| 路徑 | 內容 |
| --- | --- |
| `/` | 資料概況與專區入口 |
| `/victim`、`/victim/{topic}` | 受害者專區：安全計畫、證據保存、報警程序、校園性平、申復救濟、行動準備、時限權利、心理支持、全國性平委員 |
| `/judges`、`/judges/{topic}` | 司法現況，分四個分區：會不會定罪（各罪名定罪率）、判多重（量刑）、上訴之後（各審級結果）、法官怎麼判（承審統計） |
| `/sources` | 各資料來源的權責機關、涵蓋範圍與現況 |
| `/committee`、`/news`、`/coaches` | 校園事件統計、新聞索引、教練資訊入口 |
| `/privacy`、`/corrections`、`/licence` | 隱私政策、更正與下架、資料授權 |

路由與各頁 metadata 定義於 `app/routes.ts`。`app/site-view.tsx` 由 `usePathname()`
推導目前專區，掛載於 layout 以避免切換時 remount。

## 資料來源

| 來源 | 機關 | 產生方式 |
| --- | --- | --- |
| 校園性別事件通報統計（臺北市 109–113 學年） | 臺北市政府主計處 | 原始碼常數 |
| 全國性平調查專業人才名冊 | 教育部 | 建置時取得 |
| 公視性平新聞索引 | 公視新聞 | 每晚以監測詞查詢並累積 |
| 性平刑事裁判統計 | 司法院 | 月包批次匯入 + 每晚 API 累積 |
| 不適任教練資訊 | 運動部 | 官方入口連結 |

名冊僅保留姓名、服務單位與職稱；新聞僅保留標題、短摘要與原始連結；
裁判保留案由、案號、日期、署名法官與主文摘要，不保存判決全文。

歷史裁判來自司法院資料開放平台的每月裁判書全文包（1996 年起，共 365 個月），
在本機以 `node scripts/import-judgments.mjs <下載目錄>` 匯入。該下載端點對非瀏覽器
的請求一律回 `Entity not found`，因此壓縮檔需以瀏覽器下載，工具鏈只負責解壓與解析。
每晚的 API 僅於臺灣時間 00:00–06:00 開放，且只提供單日異動清單，無法回溯歷史；
匯入結果可上傳後以 `JUDICIAL_SOURCE_URL` 提供給 CI 作為累加的起點。

統計的判準、分母與已知限制寫在站上的「資料來源與涵蓋範圍」頁，
對應程式碼為 `app/components/judicial-method.tsx`；兩者必須一致。

`public/data/` 下的產生檔一律不進版控——名冊含兩千餘筆真實姓名，
而 git 歷史為永久保存。名冊由 `scripts/extract-talent.py` 自教育部 PDF 解析，
該解析器針對特定版本調校，不納入自動排程；解析結果放在可直接下載的位置，
以 `TALENT_SOURCE_URL` 提供給建置流程。教育部改版時在本機重跑、核對筆數後更新該檔即可。
取不到來源時寫出空名冊並在站上標示無法載入，不以舊資料或估計值填補。

## 開發

需要 Node.js `>=22.13.0`。

```bash
npm install
npm run dev

npm run data           # 產生全部資料檔
npm run data:news      # 或個別執行
npm run data:judicial
npm run data:talent

npm run build          # 靜態輸出至 out/
npm start              # 預覽 out/
npm test               # 建置後執行測試
npm run lint
```

資料腳本讀取 `.env`（見 `.env.example`）。`NEXT_PUBLIC_BASE_PATH` 控制子目錄部署，
CI 由儲存庫名稱自動推導。

## 測試

`tests/rendered-html.test.mjs` 驗證建置產物與原始碼，涵蓋各路由的 metadata 與內容、
資料管線對司法院時段與下架規則的遵循、健康檢查會使 CI 失敗、名冊排除於版控之外、
統計方法有寫在站上，以及建置產物不含任何憑證。

`tests/judgment-rules.test.mjs` 驗證裁判書的解析與分類：主文擷取不被罪名裡的
「理由」二字切斷、一部有罪一部無罪自成一類、裁定與附帶民訴不計入、刑期的中文數字
換算與應執行刑優先，以及定罪率只計第一審且排除未經實體審理的案件。

## 授權

政府開放資料依各機關授權條款使用；新聞著作權屬公視或原權利人，本站僅索引標題、
短摘要與原始連結。本站為二手整理，引用時請查核並註明原始機關公告。
流程說明不取代個案法律意見。
